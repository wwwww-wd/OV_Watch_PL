# OV_Watch 智能手表软件架构设计

> MCU: STM32F411CEU6 (Cortex-M4F, 100MHz, 512KB Flash, 128KB RAM)
> OS: FreeRTOS v10.3.1 (CMSIS-OS2)
> UI: LVGL v8.2
> Display: ST7789V 240x280 RGB565, SPI+DMA
> Touch: CST816T (软件I2C)
> 参考项目: No-Chicken/OV-Watch V2.4.5

---

## 1. 分层架构

```
┌───────────────────────────────────────────────────────┐
│                   Application Layer                    │
│  (页面逻辑: 表盘/菜单/设置/传感器/游戏/...)              │
│  对应: User/GUI_App/Screens/                           │
├───────────────────────────────────────────────────────┤
│                  GUI Framework (LVGL)                   │
│  (控件/动画/字体/输入设备)                                │
│  对应: Middlewares/LVGL/                                │
├───────────────────────────────────────────────────────┤
│                 Page Manager (页面栈)                    │
│  (Page_t + PageStack: push/pop/back)                    │
│  对应: User/Func/Src/PageManager.c                      │
├───────────────────────────────────────────────────────┤
│               Hardware Abstraction (HWInterface)        │
│  (函数指针结构体, 编译开关隔离硬件)                       │
│  对应: User/Func/Src/HWDataAccess.c                     │
├───────────────────────────────────────────────────────┤
│              FreeRTOS Tasks (任务调度)                   │
│  (8个任务 + 6个消息队列 + 1个软件定时器)                  │
│  对应: User/Tasks/                                      │
├───────────────────────────────────────────────────────┤
│                BSP Drivers (板级驱动)                    │
│  (各外设独立驱动: LCD/Touch/IMU/HR/...)                  │
│  对应: Drivers/BSP/                                     │
├───────────────────────────────────────────────────────┤
│               STM32 HAL + CubeMX 生成                   │
│  (GPIO/SPI/I2C/UART/ADC/RTC/DMA/...)                   │
│  对应: Core/ + Drivers/STM32F4xx_HAL_Driver/             │
└───────────────────────────────────────────────────────┘
```

**设计原则**: UI 代码通过 HWInterface 访问硬件，不直接调用 BSP。`HW_USE_HARDWARE=0` 时所有硬件调用变为空操作，可在 PC 上仿真 UI。

## 2. FreeRTOS 任务划分

基于原版 OV-Watch 的 12 任务方案，优化合并为 **10 个任务**：

| 优先级 | 任务名 | 周期 | 栈大小 | 职责 |
|--------|--------|------|--------|------|
| High+3 | HardwareInitTask | 一次性 | 128×10 | 初始化所有硬件+LVGL，完成后自删除 |
| Low | LvHandlerTask | 1ms | 128×24 | LVGL 主循环，检测用户活动发 IdleBreak |
| Normal | WDOGFeedTask | 500ms | 128×1 | 协作看门狗: 检查各任务签到 → 喂狗 |
| High | PowerMgrTask | 10ms | 128×16 | 低功耗管理: 灭屏 + Stop 模式 + 唤醒 |
| Normal | KeyTask | 1ms | 128×1 | 按键扫描，发 Key/Stop/IdleBreak 消息 |
| Low+1 | ScrRenewTask | 10ms | 128×10 | 页面切换: 收 Key 消息 → Page_Back |
| Low+1 | SensorTask | 500ms | 128×10 | 传感器采集: 电池/步数/心率/罗盘/天气(按需) |
| Low+2 | DataSaveTask | 100ms | 128×5 | EEPROM 存储: 设置/步数 |
| Low+2 | ChargCheckTask | 500ms | 128×5 | 充电检测: 中断标志 → 进出充电页 |
| Low+2 | MessageTask | 1000ms | 128×4 | BLE 收发: UART 命令解析+时间同步 |

**相比原版的优化**:
- 原版 IdleEnterTask + StopEnterTask → 合并为 **PowerMgrTask**（同属电源管理，共享 IdleTimerCount）
- 原版 SensorDataTask + HRDataTask + MPUCheckTask → 合并为 **SensorTask**（同属传感器采集，内部按页面分频）
- 原版 TaskTickHook 中的计时器逻辑 → 移入 **LvHandlerTask**（减少 ISR 负担）
- 原版独立的 WDOGFeedTask → 改为 **协作看门狗**（见 §2.1）

### 2.1 协作看门狗设计

**问题**: 如果看门狗是独立的高优先级任务，即使低优先级任务死锁卡死，看门狗仍能喂狗，完全失去保护作用。

**方案**: 看门狗任务本身不喂狗，而是收集各关键任务的"签到"信号，只有全部签到成功才喂狗。

```
LvHandlerTask ──签到(CH_LVGL)──┐
KeyTask       ──签到(CH_KEY)───┤
SensorTask    ──签到(CH_SENSOR)┤──► WDOGFeedTask ──全部签到?──► WDOG_Feed()
PowerMgrTask  ──签到(CH_POWER)─┤                                   │
ChargCheckTask─签到(CH_CHARG)──┘                              某个没签到?
                                                              不喂狗 → 硬件复位
```

```c
// 各任务签到宏
#define WDOG_CH_LVGL    (1 << 0)
#define WDOG_CH_KEY     (1 << 1)
#define WDOG_CH_SENSOR  (1 << 2)
#define WDOG_CH_POWER   (1 << 3)
#define WDOG_CH_CHARG   (1 << 4)
#define WDOG_CH_ALL     (WDOG_CH_LVGL | WDOG_CH_KEY | WDOG_CH_SENSOR | WDOG_CH_POWER | WDOG_CH_CHARG)

// WDOGFeedTask 逻辑
while(1) {
    wdog_checkin = 0;          // 清零所有签到
    osDelay(500);              // 等待500ms签到窗口
    if ((wdog_checkin & WDOG_CH_ALL) == WDOG_CH_ALL) {
        WDOG_Feed();           // 全部签到 → 喂狗
    }
    // 缺少任一签到 → 不喂狗 → 外部看门狗超时复位
}
```

**优先级说明**: WDOGFeedTask 优先级为 Normal，不需要抢占任何任务。它只做"检查+喂狗"，逻辑极简。

## 3. 任务间通信: 消息队列

采用原版验证过的**消息队列**方案，不使用 Mutex（嵌入式场景下消息队列更简单可靠）。

```
  ┌──────────────┐
  │   KeyTask    │──Key_MessageQueue──►┌──────────────┐
  │  (按键扫描)   │                     │ ScrRenewTask │
  │              │──Stop_MessageQueue──►│ (页面切换)    │
  └──────┬───────┘                     └──────────────┘
         │
         │──IdleBreak_MessageQueue──►┌────────────────┐
                                     │  PowerMgrTask  │
  ┌──────────────┐                   │ (灭屏/Stop模式) │
  │ LvHandlerTask│──IdleBreak_Msg──►│                │
  │ (LVGL主循环)  │                   │ Idle_MsgQueue──►│ 背光降亮度
  └──────────────┘                   │ Stop_MsgQueue──►│ 进入Stop
                                     └────────────────┘
  ┌──────────────┐
  │  SensorTask  │──HomeUpdata_Msg──► 刷新主页数据
  │ (传感器采集)  │──DataSave_Msg───►┌──────────────┐
  └──────────────┘                   │ DataSaveTask │
                                     │ (EEPROM存储)  │
                                     └──────────────┘
```

### 消息队列定义

| 队列名 | 深度 | 元素大小 | 生产者 | 消费者 | 用途 |
|--------|------|---------|--------|--------|------|
| Key_MessageQueue | 1 | 1B | KeyTask | ScrRenewTask | 按键值(1=key1, 2=key2) |
| Idle_MessageQueue | 1 | 1B | IdleTimer | PowerMgrTask | 灭屏指令 |
| Stop_MessageQueue | 1 | 1B | KeyTask/MPU | PowerMgrTask | 进入Stop模式 |
| IdleBreak_MessageQueue | 1 | 1B | KeyTask/LvHandler/触摸 | PowerMgrTask | 亮屏/重置计时 |
| HomeUpdata_MessageQueue | 1 | 1B | PowerMgrTask/SensorTask | SensorTask | 刷新主页数据 |
| DataSave_MessageQueue | 2 | 1B | SensorTask | DataSaveTask | 触发EEPROM写入 |

### 软件定时器

| 定时器 | 周期 | 回调 | 用途 |
|--------|------|------|------|
| IdleTimer | 100ms | IdleTimerCallback | 超时计数: lTime→灭屏, tTime→Stop |

### 3.2 数据流: HWInterface 全局结构体

消息队列只负责**任务间通知**，实际数据通过 `HWInterface` 全局结构体传递。这是一种**「单写多读无锁」**模式。

```
┌──────────────────────────────────────────────────────────────┐
│                   HWInterface (全局结构体)                     │
│  .Power.power_remain = 85                                    │
│  .AHT21.temperature  = 26       .AHT21.humidity    = 67      │
│  .IMU.Steps          = 1234     .HR_meter.HrRate   = 72      │
│  .Ecompass.direction = 180     .Barometer.altitude = 1200    │
└──────────┬──────────────┬──────────────┬─────────────────────┘
           │              │              │
   写入: SensorTask       │      读取: DataSaveTask
           │              │              │
           │       读取: LVGL UI         │
           │              │              │
           ▼              ▼              ▼
```

**写入方 — SensorTask**:
```c
// 主页数据 (HomeUpdata 消息触发): 仅电池
HWInterface.Power.power_remain = HWInterface.Power.BatCalculate();

// 天气页面 (仅在天气页时采样): 温湿度
if (当前在天气页面) {
    HWInterface.AHT21.GetHumiTemp(&humi, &temp);
    HWInterface.AHT21.temperature = (int8_t)temp;
    HWInterface.AHT21.humidity = (int8_t)humi;
}
```

**写入方 — 主页 LVGL 定时器** (步数直接读取，不经过 SensorTask):
```c
// HomePage_timer_cb 中，1s 周期
uint16_t steps = HWInterface.IMU.GetSteps();  // 直接读 IMU
```

**读取方1 — LVGL UI** (天气页面直接读取):
```c
lv_label_set_text_fmt(temp_label, "%d°C", HWInterface.AHT21.temperature);
lv_label_set_text_fmt(humi_label, "%d%%", HWInterface.AHT21.humidity);
```

**读取方2 — DataSaveTask** (收到通知后读取 → 写 EEPROM):
```c
uint16_t steps = HWInterface.IMU.GetSteps();
dat[0] = nowdate.Date;
dat[1] = steps >> 8;
dat[2] = steps & 0xff;
SettingSave(dat, 0x20, 3);
```

**为什么不需要 Mutex**:
- 每个字段只有一个任务写入（SensorTask 写，UI/DataSaveTask 读）
- 数据类型都是 `uint8_t` / `uint16_t`（单次内存访问，天然原子）
- 即使读到旧值，传感器数据本身就是周期性更新的，无影响

## 4. 低功耗状态机

```
            ┌──────────────────────────────────────────┐
            │              Running 正常运行              │
            │  LCD亮, LVGL跑, 传感器采集                  │
            └─────────┬────────────────────┬───────────┘
                      │                    │
              lTime超时(IdleMsg)     tTime超时(StopMsg)
              (用户可配置)           (用户可配置)
                      │                    │
                      ▼                    ▼
            ┌─────────────────┐  ┌─────────────────────┐
            │   Idle 灭屏     │  │   Stop 低功耗模式     │
            │  LCD背光=5%     │  │  关LCD/Touch/UART     │
            │  LVGL继续跑     │  │  HAL_PWR_EnterSTOP   │
            └────────┬────────┘  │  等待唤醒中断         │
                     │           └──────────┬──────────┘
             IdleBreak(按键/触摸)            │
                     │           唤醒源: KEY/充电/抬腕
                     ▼                      │
            回到 Running             ┌──────▼──────┐
            背光恢复                 │  唤醒判断    │
                                    │  KEY/充电?   │──是──► 回到Running
                                    │  抬腕?       │──是──► 回到Running
                                    │  其他?       │──否──► 回到Stop
                                    └─────────────┘
```

**低功耗效果**:
- Idle 模式: 背光降至 5%，功耗约 15mA → 5mA
- Stop 模式: STM32 Stop 模式，功耗约 2mA → 50μA
- 抬腕检测: MPU6050 中断唤醒，无需持续轮询

## 7. 任务详细设计

### 7.1 HardwareInitTask (一次性任务)

```
启动 → vTaskSuspendAll()
  → RTC 初始化 (备份寄存器保护, 见下文)
  → UART DMA 启动
  → PWM 背光启动
  → delay_init()
  → Power.Init()
  → Key_Port_Init()
  → 传感器初始化 (AHT21/Ecompass/Barometer/IMU/HR_meter, 各重试3次)
  → EEPROM 初始化 + 读取设置
  → BLE.Init() + Disable()
  → Touch CST816 初始化
  → LCD 初始化 + 开机画面
  → LVGL 初始化 (lv_init/lv_port_disp/lv_port_indev/ui_init)
→ xTaskResumeAll()
→ vTaskDelete(NULL) 自删除
```

**RTC 备份寄存器保护**:

STM32F411 的备份寄存器 (BKP) 由 VBAT 供电，软件复位和掉电不丢失（需纽扣电池）。用备份寄存器存 magic number，避免每次烧录重置时间：

```c
// rtc.c — CubeMX 预留的 USER CODE 区域
/* USER CODE BEGIN Check_RTC_BKUP */
#define RTC_MAGIC 0x5A5A

if (HAL_RTCEx_BKUPRead(&hrtc, RTC_BKP_DR1) != RTC_MAGIC) {
    // 首次烧录: 设置默认时间 + 写入 magic
    sTime.Hours = 12; sTime.Minutes = 0; sTime.Seconds = 0;
    HAL_RTC_SetTime(&hrtc, &sTime, RTC_FORMAT_BCD);
    sDate.Year = 26; sDate.Month = 6; sDate.Date = 15;
    HAL_RTC_SetDate(&hrtc, &sDate, RTC_FORMAT_BCD);
    HAL_RTCEx_BKUPWrite(&hrtc, RTC_BKP_DR1, RTC_MAGIC);
} else {
    // 已有时间，跳过设置
}
/* USER CODE END Check_RTC_BKUP */
```

| 场景 | BKP_DR1 | 行为 |
|------|---------|------|
| 首次烧录 | 0xFFFF (空) | 设置默认时间，写入 0x5A5A |
| 后续烧录/复位 | 0x5A5A | 跳过时间设置，保留当前时间 |
| 更换纽扣电池 | 0xFFFF (丢失) | 重新设置时间 |

### 7.2 LvHandlerTask (常驻)

```c
while(1) {
    // 检测用户活动（触摸/动画），发 IdleBreak 消息
    if (lv_disp_get_inactive_time(NULL) < 1000) {
        osMessageQueuePut(IdleBreak_MessageQueue, ...);
    }
    lv_task_handler();  // LVGL 主循环
    osDelay(1);         // 1ms 周期
}
```

### 7.3 PowerMgrTask (低功耗管理)

```c
// IdleEnter 部分: 接收灭屏/亮屏消息
if (osMessageQueueGet(Idle_MessageQueue, ...) == osOK) {
    LCD_Set_Light(5);  // 背光降至5%
}
if (osMessageQueueGet(IdleBreak_MessageQueue, ...) == osOK) {
    IdleTimerCount = 0;
    LCD_Set_Light(ui_LightSliderValue);  // 恢复亮度
}

// StopEnter 部分: 收到Stop消息 → 关外设 → EnterSTOPMode → 唤醒 → 恢复
if (osMessageQueueGet(Stop_MessageQueue, ...) == osOK) {
    // 关 LCD/Touch/UART
    // vTaskSuspendAll + 关 SysTick
    // HAL_PWR_EnterSTOPMode()
    // --- 休眠中 ---
    // 唤醒: 恢复时钟 + 判断唤醒源
    // KEY/充电/抬腕 → 恢复运行; 否则 → 回到Stop
    // 恢复 LCD/Touch/UART
    // 发 HomeUpdata 消息刷新主页
}
```

### 7.4 IdleTimerCallback (软件定时器, 100ms)

```c
void IdleTimerCallback(void *argument) {
    IdleTimerCount++;
    if (IdleTimerCount == ui_LTimeValue * 10) {  // 灭屏超时
        osMessageQueuePut(Idle_MessageQueue, ...);
    }
    if (IdleTimerCount == ui_TTimeValue * 10) {  // Stop超时
        IdleTimerCount = 0;
        osMessageQueuePut(Stop_MessageQueue, ...);
    }
}
```

### 7.5 SensorTask (传感器采集)

```c
while(1) {
    // 主页数据刷新 (收到 HomeUpdata 消息时, 仅电池)
    if (osMessageQueueGet(HomeUpdata_MessageQueue, ...) == osOK) {
        HWInterface.Power.power_remain = HWInterface.Power.BatCalculate();

    // 天气页面: 温湿度采集 (AHT21 按需启停)
    if (Page_Get_NowPage()->page_obj == &ui_WeatherPage) {

            HWInterface.AHT21.temperature = (int8_t)temp;
            HWInterface.AHT21.humidity = (int8_t)humi;
        }

    // 心率页面: 50ms 采样
    if (Page_Get_NowPage()->page_obj == &ui_HRPage) {

    }

    // 罗盘页面: 磁力计+气压计
    if (Page_Get_NowPage()->page_obj == &ui_CompassPage) {
        LSM303DLH_Wakeup();
        // 读取加速度+磁场 → 计算方位角
        // 读取气压 → 计算海拔
    }

    // 抬腕检测: 300ms 周期
    if (HWInterface.IMU.wrist_is_enabled) {
        if (MPU_isHorizontal()) wrist_state = UP;
```

### 7.6 DataSaveTask (EEPROM 存储)

EEPROM 数据布局：

| 地址 | 内容 | 说明 |
|------|------|------|
| 0x00 | 0x55 | 校验标志 |
| 0x01 | 0xAA | 校验标志 |
| 0x10 | wrist_is_enabled | 抬腕检测开关 |
| 0x11 | ui_APPSy_EN | APP同步开关 |
| 0x20 | Last Save Day | 上次保存日期(1-31) |
| 0x21 | Steps High | 步数高字节 |
| 0x22 | Steps Low | 步数低字节 |

**启动时读取逻辑** (HardwareInitTask 中)：

```c
EEPROM_Init();
if(!EEPROM_Check())
{
    uint8_t recbuf[3];
    SettingGet(recbuf, 0x20, 3);
    
    RTC_DateTypeDef nowdate;
    HAL_RTC_GetDate(&hrtc, &nowdate, RTC_FORMAT_BIN);
    
    if(recbuf[0] == nowdate.Date)
    {
        // 同一天: 从EEPROM恢复步数
        uint16_t steps = (recbuf[1] << 8) | recbuf[2];
        dmp_set_pedometer_step_count((unsigned long)steps);
    }
    else
    {
        // 新的一天: 步数清零
        dmp_set_pedometer_step_count(0);
    }
}
```

**运行时保存逻辑** (DataSaveTask 中)：

```c
void DataSaveTask(void *argument)
{
    while(1)
    {
        uint8_t Datastr = 0;
        if(osMessageQueueGet(DataSave_MessageQueue, &Datastr, NULL, 1) == osOK)
        {
            uint8_t dat[3];
            
            RTC_DateTypeDef nowdate;
            HAL_RTC_GetDate(&hrtc, &nowdate, RTC_FORMAT_BIN);
            
            uint16_t steps = HWInterface.IMU.GetSteps();
            dat[0] = nowdate.Date;
            dat[1] = (steps >> 8) & 0xFF;
            dat[2] = steps & 0xFF;
            SettingSave(dat, 0x20, 3);
        }
        osDelay(100);
    }
}
```

**采样策略: 按需启停，省功耗**

| 数据 | 采样时机 | 传感器状态 | 原因 |
|------|---------|-----------|------|
| 时钟(时:分) | LVGL 定时器 500ms | RTC 常开 | 主页实时显示 |
| 日期(年-月-日) | LVGL 定时器 500ms | RTC 常开 | 日期变化时更新 |
| 步数 | LVGL 定时器 500ms | IMU 常开 | 走路时需实时反馈 |
| 电池 | HomeUpdata 消息触发 | ADC 常开 | 变化慢，低频刷新 |
| 温湿度 | 仅在天气页面 | AHT21 按需启停 | 功耗敏感 |
| 心率 | 仅在心率页面 + 独立50ms | EM7028 按需启停 | 精确采样 |
| 罗盘/气压 | 仅在罗盘页面 | LSM303/SPL06 按需启停 | 磁力计功耗高 |

**主页布局**:

```
┌─────────────────────────┐
│                         │
│        12:45            │ ← 时钟 (大字体)
│      2026-06-16         │ ← 日期 (小字体)
│                         │
│   🏠 1234   🔋 85%      │ ← 步数 + 电量
│                         │
└─────────────────────────┘
```

**主页 LVGL 定时器设计**:

主页使用 LVGL 内置定时器刷新数据，无需额外 FreeRTOS 任务：

```c
// 主页创建时启动 (ui_HomePage.c)
ui_HomePageTimer = lv_timer_create(HomePage_timer_cb, 500, NULL);

static void HomePage_timer_cb(lv_timer_t * timer) {
    // 时钟: 500ms 刷新
    HWInterface.RealTimeClock.GetTimeDate(&DateTime);
    if (ui_TimeMinuteValue != DateTime.Minutes) { ... }
    
    // 日期: 日期变化时更新
    if (ui_DateValue != DateTime.Date) { ... }
    
    // 步数: 500ms 刷新
    uint16_t steps = HWInterface.IMU.GetSteps();
    lv_label_set_text_fmt(steps_label, "%d", steps);
}
```

离开主页后 LVGL 定时器自动销毁，不浪费资源。

## 9. 数据流示例

### 9.1 按键 → 页面切换

```
KeyTask [1ms轮询]
  │ KeyScan() 检测到按键
  │ keystr=1 → osMessageQueuePut(Key_MessageQueue)
  │            osMessageQueuePut(IdleBreak_MessageQueue)
  ▼
ScrRenewTask [10ms轮询]
  │ osMessageQueueGet(Key_MessageQueue) → keystr=1
  │ Page_Back() → 弹出栈顶，加载上一页
  │ 如果回到MenuPage → 传感器休眠
  ▼
LvHandlerTask [1ms轮询]
  │ lv_task_handler() 渲染新页面
  │ SPI DMA flush → LCD 显示
```

### 9.2 超时灭屏 → Stop模式

```
IdleTimerCallback [100ms定时器]
  │ IdleTimerCount++
  │ 到达 lTime → osMessageQueuePut(Idle_MessageQueue)
  ▼
PowerMgrTask
  │ 收到 Idle_MessageQueue → LCD_Set_Light(5)
  │ IdleTimerCount 继续++
  │ 到达 tTime → osMessageQueuePut(Stop_MessageQueue)
  ▼
PowerMgrTask
  │ 收到 Stop_MessageQueue
  │ 关LCD/Touch/UART → HAL_PWR_EnterSTOPMode()
  │ ... 休眠 ...
  │ KEY中断唤醒 → 恢复时钟/外设
  │ 发 HomeUpdata_MessageQueue → 刷新主页
```
